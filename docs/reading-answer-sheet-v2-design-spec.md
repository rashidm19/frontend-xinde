## 1. Executive Summary

The Reading Answer Sheet V2 page displays detailed IELTS reading practice results with rich question-by-question feedback. This design specification defines a visually striking, animated interface that helps IELTS students review their reading practice performance with engagement and clarity.

### Design Goals
1. **Educational Clarity** - Make correct/incorrect answers immediately recognizable
2. **Progressive Disclosure** - Allow users to drill down from summary â†’ parts â†’ blocks â†’ questions
3. **Engagement** - Use animations and visual hierarchy to maintain focus
4. **Accessibility** - WCAG 2.1 AA compliant with reduced motion support
5. **Mobile-First** - Fully responsive with touch-optimized interactions

## 8. Accessibility Requirements

### 8.1 WCAG 2.1 AA Compliance

- **Color Contrast:** All text meets 4.5:1 ratio minimum
- **Focus Indicators:** Visible focus rings on all interactive elements
- **Keyboard Navigation:** Full keyboard accessibility for all features
- **Screen Reader:** Proper ARIA labels and semantic HTML
- **Reduced Motion:** Animation disable option respected

### 8.2 ARIA Implementation

```html
<!-- Accordion pattern -->
<button aria-expanded="true" aria-controls="content-1">
  Block Header
</button>
<div id="content-1" role="region">
  Content
</div>

<!-- Tab pattern for parts -->
<div role="tablist" aria-label="Reading parts">
  <button role="tab" aria-selected="true" aria-controls="part-1">Part 1</button>
  <button role="tab" aria-selected="false" aria-controls="part-2">Part 2</button>
  <button role="tab" aria-selected="false" aria-controls="part-3">Part 3</button>
</div>
<div role="tabpanel" id="part-1">Content</div>

<!-- Status announcements -->
<span className="sr-only">Question 16, Incorrect. Your answer: False. Correct answer: True.</span>
```

### 8.3 Focus Management

- Focus trapped within open modals/sheets
- Focus returns to trigger when modal closes
- Skip links for main content navigation
- Logical tab order through all controls

## 10. Performance Considerations

### 10.1 Virtualization

For parts with many questions (>20), consider virtual scrolling:
- Render only visible questions
- Use intersection observer for lazy loading
- Maintain scroll position on tab switch

### 10.2 Image Optimization

- Diagrams: Use Next.js Image component with lazy loading
- Fallback placeholder for slow connections
- Appropriate sizing constraints

### 10.3 Animation Performance

- Use `transform` and `opacity` for animations (GPU accelerated)
- Avoid layout-triggering properties in animations
- Use `will-change` sparingly for known animated elements
